# 第七版：AI 题目生成联网增强方案

## 1. 目标

在 AI 生成候选题时，允许用户选择是否启用联网搜索增强。

启用后，系统会结合：

- 当前题目
- 用户本次答案
- AI 讲题助手对话
- AI 主观题评分与评分追问
- 联网搜索结果和网页相关片段

生成新的候选题，从而提升题目的工程背景、知识覆盖和变式质量。

联网增强不是默认强制行为。用户不勾选时，仍使用原有的本地上下文生成流程。

## 2. 为什么不做完整 tool-calling

当前项目的 AI 调用模式是：

```text
后端组装 prompt
调用大模型
模型直接返回最终 JSON
后端解析 JSON
```

完整 tool-calling 需要额外实现：

```text
模型返回 tool_call
后端解析 tool_call
后端执行工具
后端把 tool_result 回填给模型
模型继续请求工具或返回最终答案
```

这套循环可以做，但对当前“联网增强生成题目”来说偏重。

本阶段采用方案 A：

```text
模型生成搜索句
后端调用 AnySearch
后端提取网页正文
后端用 BM25 抽取相关片段
最终出题模型读取联网片段并生成候选题
```

这样模型仍参与“搜什么”的决策，但工具执行由后端固定流程完成，稳定性更高。

## 3. 用户交互

AI 生成候选题弹窗中增加：

```text
[ ] 联网搜索增强
```

说明文案：

```text
启用后会先搜索相关资料，再结合本题上下文生成候选题。联网资料仅作参考。
```

前端请求增加字段：

```json
{
  "use_web_search": true
}
```

不启用时不调用 AnySearch。

## 4. 流程设计

### 4.1 搜索句生成

后端先让模型根据本题上下文生成 2-4 条搜索句。

搜索句可以是关键词，也可以是完整问题句，不强制是短词。

输出格式：

```json
{
  "queries": [
    {
      "query": "MCP tool_result 状态写回 Agent 工具调用",
      "intent": "查 MCP 工具调用结果如何回传给模型"
    }
  ]
}
```

约束：

- 最多 4 条。
- 每条必须有 `query` 和 `intent`。
- 搜索句应围绕当前题目、用户答题缺口、AI 对话暴露出的薄弱点。
- 不要生成过泛的搜索句，例如“AI Agent”。

### 4.2 搜索结果获取

对每条搜索句调用 AnySearch `search`：

```text
每条搜索句最多取 2 条搜索结果
```

如果有 X 条搜索句，则最多得到 `2X` 个候选网页。

第一版使用通用搜索，不接入 AnySearch 垂直领域搜索。

### 4.3 网页选择

从所有候选网页中选择 2-3 个 URL 进行正文提取。

选择规则：

- 优先保证不同搜索意图都被覆盖。
- 每个搜索句优先选一个最高分结果。
- 同一域名不要占满全部名额。
- 标题和摘要明显无关、广告、下载站、营销页降权。

第一版由代码规则完成，不让模型二次选择。

### 4.4 网页正文提取

对选中的 URL 调用 AnySearch `extract`。

`extract` 返回 Markdown 正文，可能包含导航、广告、目录和无关段落，因此不能直接全文塞给出题模型。

### 4.5 相关片段抽取

不使用大模型对网页全文做摘要，避免重复消耗 token。

后端使用代码进行片段抽取：

```text
Markdown 正文
↓
规则清洗
↓
按标题、空行、列表切分 chunk
↓
用 BM25 根据搜索句、题目、考察点、AI 对话关键词排序
↓
取 top chunk
```

推荐限制：

- 最多提取网页：3 个
- 每个网页最多片段：2 个
- 每个片段最多：1000 字
- 联网资料总字数：4000-6000 字

片段需要保留来源信息：

```text
来源标题
来源 URL
搜索意图
搜索摘要
标题路径
原文片段
```

这样即使片段来自不同网页，模型也能理解每段的上下文。

## 5. 插入 prompt 的方式

联网资料放在本地学习上下文之后：

```text
当前题目上下文
用户答案上下文
AI 评分上下文
AI 对话上下文
题库已有分类全集
联网参考资料
生成要求
```

联网参考资料格式：

```text
联网参考资料：
说明：以下内容来自搜索结果和网页原文片段，只作为参考，不代表绝对正确。

[资料 1]
搜索意图：...
来源标题：...
来源 URL：...
搜索摘要：...
相关片段：
1. 标题路径：...
原文片段：...
```

## 6. 信任边界

必须明确告诉最终出题模型：

```text
联网资料可能错误、过时、片面或与题库上下文冲突。
联网资料只用于补充工程背景、术语和场景。
如果联网资料与当前题目标准答案、题库知识点或用户学习上下文冲突，以当前题库上下文为准。
不要直接复制网页原文生成题干、选项或标准答案。
生成题目必须可以脱离联网资料独立作答。
```

## 7. 降级策略

联网搜索失败时，不应该导致整个题目生成失败。

降级规则：

- 搜索句生成失败：直接使用普通生成。
- AnySearch 搜索失败：直接使用普通生成。
- 部分网页 extract 失败：跳过该网页。
- 没有可用片段：直接使用普通生成。

前端可以显示简短提示，但候选题生成仍应继续。

## 8. 后端模块建议

新增：

```text
backend/app/services/search/anysearch_client.py
backend/app/services/search/web_context_service.py
```

职责：

```text
anysearch_client.py
- 调用 AnySearch JSON-RPC API
- search
- extract

web_context_service.py
- 生成搜索句
- 搜索候选网页
- 选择 URL
- 提取网页正文
- BM25 抽片段
- 格式化联网参考资料
```

现有生成服务：

```text
backend/app/services/llm/ai_question_generation_service.py
```

接入方式：

```text
if payload.use_web_search:
    web_context = build_web_context(...)
else:
    web_context = None

_generation_messages(..., web_context)
```

## 9. 配置

后端环境变量：

```text
ANYSEARCH_API_KEY=可选
ANYSEARCH_ENABLED=true
ANYSEARCH_ENDPOINT=https://api.anysearch.com/mcp
```

没有 API Key 时可走匿名访问。匿名额度不足时自动降级为普通生成。

## 10. 当前阶段边界

本阶段不做：

- 完整 tool-calling loop
- AnySearch 垂直领域搜索
- 搜索结果长期缓存
- 网页全文入库
- 前端展示联网来源详情

后续可以在候选题确认页展示“联网参考来源”，但第一版先只用于提升生成质量。
