你现在需要帮我完整设计并实现一个“Agent 应用层岗位题库刷题系统”。

这是一个本地运行的前后端分离 Web 项目，不是命令行工具。项目目标是把我已经整理好的题库 Markdown 文件解析成结构化题库，然后提供一个美观的网页客户端，供用户刷题、查看解析、复习错题和查看统计。

## 一、已有题库文件

题库文件已经放在项目根目录下面，文件名是：

```text
Agent 应用层岗位题库.md
```

你需要优先读取这个文件，并以它作为第一版题库解析和功能测试的数据来源。

注意：

1. 不要让我重新粘贴题库内容。
2. 不要假设题库只有单选、多选、判断。
3. 你需要根据这个 Markdown 文件的实际内容设计解析器。
4. 如果发现某些题目格式不规范，需要在导入预览或日志中明确指出，而不是静默丢弃。
5. 系统第一版只按照题库里的“标准答案”判分，不要自行改题目答案。

## 二、项目目标

我要做的是一个可以长期维护和扩展的题库刷题系统。

整体流程如下：

```text
根目录题库文件 Agent 应用层岗位题库.md
        ↓
Markdown 题库解析器
        ↓
结构化题库数据
        ↓
SQLite 本地数据库
        ↓
FastAPI 后端 API
        ↓
React 网页客户端
        ↓
用户刷题、判分、看解析、复习错题、看统计
```

核心目标：

1. 能解析根目录下的 `Agent 应用层岗位题库.md`
2. 能将题库导入 SQLite
3. 能在网页端展示题库
4. 能按题型、难度、标签筛选题目
5. 能随机刷题
6. 能提交答案并自动判分
7. 能展示标准答案、详细解析、考察点、常见错误、面试延伸追问、评分标准
8. 能记录答题历史
9. 能查看错题
10. 能查看基础统计
11. 架构要方便后续扩展更多题型

## 八、题库解析要求

题库文件是 Markdown 格式，可能包含类似结构：

```text
### Part 2-015｜单选题｜进阶｜高仿真题

**题目：** 围绕 工作流 Agent，以下哪种设计风险最大？

**选项/材料：**
- A. 只扩大模型上下文窗口，尽量把所有数据塞给模型
- B. 建立可观测链路，定位实际失效环节，再做针对性修复
- C. 直接把 temperature 调到最高，让模型探索更多答案
- D. 取消工具/检索约束，让模型自由推理

**标准答案：** B

**详细解析：** ...

**考察点：** ...

**常见错误：** ...

**面试延伸追问：** ...

**评分标准：** ...
```

也可能是：

```text
Part 2-016｜多选题｜基础必会｜高仿真题
```

解析器需要兼容：

1. 标题前有 `###`
2. 标题前没有 `###`
3. 字段名有 Markdown 加粗
4. 字段名没有 Markdown 加粗
5. 中文冒号 `：`
6. 英文冒号 `:`
7. 空行数量不固定
8. 分隔符是中文竖线 `｜`
9. 选项写成 `- A. xxx`
10. 选项写成 `A. xxx`
11. 判断题可能没有 A/B/C/D 选项
12. 填空题可能没有选项
13. 简答题可能没有选项
14. 多选题标准答案可能是 `A、B、D`
15. 多选题标准答案可能是 `A,B,D`
16. 多选题标准答案可能是 `ABD`
17. 判断题答案可能是 `正确` / `错误`
18. 某些字段可能缺失，需要记录 warning
19. 严重缺失题干或标准答案时，需要记录 import error

## 十、判分器架构设计

后端需要实现可扩展 AnswerChecker。

不要把所有题型判分写在一个函数里。

建议结构：

```text

  base.py
  choice_checker.py
```

### BaseAnswerChecker

负责定义统一接口：

```python
class BaseAnswerChecker:
    def normalize_standard_answer(self, raw_answer: Any) -> Any:
        ...

    def normalize_user_answer(self, raw_answer: Any) -> Any:
        ...

    def check(self, question: Question, user_answer: Any) -> CheckResult:
        ...
```

### SingleChoiceChecker

支持：

```text
A
a
 A 
```

统一归一化为：

```text
A
```

### MultipleChoiceChecker

支持：

```text
A、B、D
A,B,D
ABD
a b d
```

统一归一化为集合：

```python
{"A", "B", "D"}
```

多选题必须完全一致才算正确，多选、少选、错选都算错误。

### TrueFalseChecker

支持：

```text
正确
错误
对
错
true
false
T
F
是
否
```

统一归一化为 boolean 或固定枚举。

### FillBlankChecker

第一版只做可控规则判分，不做语义判分。

支持：

1. 去掉首尾空格
2. 大小写归一化
3. 中英文符号归一化
4. 多个可接受答案

例如标准答案：

```text
上下文工程 / context engineering / Context Engineering
```

用户填其中一个即可算正确。

### SubjectiveChecker

用于简答题、问答题、论述题。

第一版不要自动判分。

提交后返回：

```json
{
  "is_correct": null,
  "requires_self_review": true,
  "standard_answer": "...",
  "explanation": "...",
  "scoring_standard": "..."
}
```

前端让用户选择：

```text
我答对了
我部分答对
我答错了
```

然后记录自评结果。

## 十八、运行方式

后端启动方式：

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
```

前端启动方式：

```bash
cd frontend
npm install
npm run dev
```

后端测试：

```bash
cd backend
pytest
```

默认访问地址：

```text
前端：http://localhost:5173
后端：http://localhost:8000
API 文档：http://localhost:8000/docs
```
