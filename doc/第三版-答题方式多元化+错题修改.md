## 三、当前主要问题

### 问题 1：现在只有随机练习

当前刷题页基本是随机出题。这样有几个问题：

1. 用户无法按导入顺序从第一题开始刷。
2. 用户无法按题库原始顺序连续刷题。
3. 用户无法按某个考察点专项练习。
4. 用户无法按某个题目方向专项练习。
5. 用户想复习错题时，只能去随机模式里碰运气，体验很差。

### 问题 2：错题复习页不能直接进入刷题

错题页现在只是展示错题信息，但用户点击后不能直接进入该错题进行复习或重新作答。

需要改成：

```text id="yzykkx"
错题列表
  ↓ 点击某道错题的“重新练习”
进入刷题页
  ↓ 当前题目就是这道错题
提交答案
  ↓ 展示解析
  ↓ 下一题继续错题专项
```

### 问题 3：错题过多时展示方式不合理

错题多了以后不能一次性全部显示。需要分页、筛选、排序。

### 问题 4：题目需要有“方向”字段

现在题目有考察点，但用户后续需要自己维护题目的“方向”，例如：

```text id="43rxrz"
工作流 Agent
上下文工程
工具 schema
RAG
安全权限
多 Agent
记忆系统
结构化输出
可观测性
Agent 评估
生产排障
```

用户修改题目时，应该可以修改这个方向。后续可以根据方向做专项练习。

## 五、数据库与模型修改

### 1. Question 增加 import_order 字段

需要给 Question 增加：

```python id="sihqdn"
import_order: int | None
```

作用：

* 保存题目在原始 Markdown 导入时的顺序
* 支持按导入顺序刷题
* 支持题库列表按原始顺序排序

要求：

1. 新导入题目时写入 import_order。
2. 已有数据库需要轻量迁移，不要让我删库。
3. 对已有题目进行 backfill：

   * 优先按 part_id 自然排序，例如 Part 1-001、Part 1-002、Part 2-015。
   * 如果 part_id 无法排序，再按 created_at / id 稳定排序。
4. import_order 不能因为用户修改题目内容而改变。

### 2. Question 增加 directions 字段

需要给 Question 增加：

```python id="2uioxi"
directions: list[str]
```

作用：

* 表示题目所属方向
* 用户可以手动维护
* 后续用于专项练习和统计

实现方式：

* SQLite 中可以用 Text 存 JSON 字符串。
* 读写时封装序列化和反序列化。
* 需要中文注释说明为什么这样存。

初始化策略：

1. 新导入题目时，可以先从 exam_points 中提取方向。
2. 如果无法提取，directions 为空。
3. 不要强行乱归类。
4. 用户手动修改后的 directions 以后优先使用用户修改结果。

### 3. Attempt 可以保留现有结构

Attempt 已经有 question_version 和 question_snapshot，可以继续使用。

如果需要支持专项练习统计，可以后续再扩展，不要本次过度设计。
