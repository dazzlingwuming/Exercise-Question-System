reference_answer: |
  基于规则的护栏使用静态逻辑（正则、黑名单等）来识别违规内容。其优点是实现简单、执行速度快且可预测，但缺点是不具备语义理解，容易漏检绕过变种。  
  基于模型的护栏使用训练好的LLM或分类器对输入/输出进行语义评估。它们能捕捉上下文和隐晦违规，提高覆盖率，但成本较高、响应延迟大，且可能产生误判。  
  适用场景上，规则护栏适用于漏洞可枚举、易于量化的违规（如固定敏感词），而模型护栏适用于复杂场景或需要理解上下文的情况。通常实际系统会结合两者：规则快速过滤常见问题，模型捕捉复杂风险，以平衡性能和准确性。  

answer_keywords:  
  - 正则匹配  
  - 关键字过滤

pseudocode: |
  def filter_sensitive(text):
      patterns = {"email": r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b",
                  "phone": r"\b\d{3}[-\s]?\d{4}[-\s]?\d{4}\b"}  # 示例模式

          matches = regex.find_all(text)  # 假设有find_all方法
          for match in matches:
              placeholder = "[REDACTED_" + type + "]"
              text = text.replace(match, placeholder)
      return text

complexity:  
  time: O(n * k)  # n为文本长度, k为敏感模式数量  
  space: O(n)
